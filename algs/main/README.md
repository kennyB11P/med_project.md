## Описание файлов

### `dataset_downloading.ipynb`

Скачивает исходный датасет отзывов Wildberries с Hugging Face и сохраняет его на Google Drive.

Ноутбук загружает датасет `nyuuzyou/wb-feedbacks` в streaming-режиме, то есть не держит весь датасет в памяти. Данные читаются постепенно, чанками по `100 000` строк. Для каждого чанка сохраняется две версии:

- сырая версия без изменений;
- очищенная версия, где обрабатывается текст отзыва.

Очистка текста включает:

- замену `NaN` на пустую строку;
- удаление переносов строк и табов;
- схлопывание лишних пробелов;
- обрезку слишком длинных отзывов до `5000` символов;
- удаление пустых отзывов.

Основной результат:

- `data/raw/wb_feedbacks_full/raw_part_*.parquet`
- `data/interim/wb_feedbacks_clean_full/clean_part_*.parquet`

Файл нужен как первый этап пайплайна: он скачивает и подготавливает полную базу отзывов, из которой дальше берутся выборки для разметки, кластеризации и обучения модели.

### `makes_200_000_random_samples_from_100_chunks.ipynb`

Создает случайную выборку отзывов из уже скачанных очищенных parquet-чанков.

Ноутбук не скачивает данные заново и не вызывает LLM. Он берет файлы `clean_part_*.parquet` из директории с очищенными отзывами, случайно выбирает `100` чанков с фиксированным `RANDOM_SEED = 42`, а затем из каждого выбранного чанка берет до `2000` отзывов.

Итоговая выборка получается размером до `200 000` отзывов.

Основной результат:

- `sample_100_chunks_2000_each.parquet`
- `sample_100_chunks_2000_each.csv`
- `selected_chunks.csv`
- `rows_per_chunk.csv`
- `metadata.json`
- промежуточные файлы `sampled_parts/sample_part_*.parquet`

Файл нужен для формирования большой, но управляемой случайной выборки из полной базы отзывов. Эту выборку потом можно использовать для разметки через LLM или других экспериментов.

### `data_markup.ipynb`

Размечает отзывы через LLM API и собирает сбалансированный датасет по классам.

Ноутбук берет подготовленную выборку отзывов из parquet-файла и отправляет отзывы в модель `qwen/qwen3-32b` через API. Для каждого отзыва модель должна вернуть multi-label разметку: один отзыв может относиться к одному или нескольким классам.

Основная логика:

- данные читаются из заранее подготовленного parquet-файла;
- отзывы отправляются в LLM по одному;
- ответ модели нормализуется и проверяется;
- сохраняются только разрешенные классы;
- пример добавляется в датасет только если он пополняет хотя бы один еще не заполненный класс;
- для каждого класса собирается заданное число примеров;
- поддерживается продолжение разметки из уже существующего CSV, чтобы не начинать заново после остановки.

Основной результат:

- `balanced_50_per_class_random_chunks.csv`
- `balanced_50_per_class_random_chunks.parquet`
- `balanced_50_per_class_random_chunks_final.csv`
- `balanced_50_per_class_random_chunks_class_status.csv`

Файл нужен для первичного сбора размеченного датасета отзывов, где классы представлены более-менее сбалансированно.

### `create_golden_set_by_class.ipynb`

Создает golden set для ручной проверки качества разметки и оценки модели.

Ноутбук объединяет размеченные данные из двух источников:

- выборка, размеченная из случайных чанков;
- выборка, добранная из HDBSCAN-кластеров.

После объединения он разбирает multi-label разметку и для каждого класса создает отдельный CSV-файл. Если по классу есть `100` или больше примеров, выбирается случайные `100`; если меньше — сохраняются все доступные примеры.

Один и тот же отзыв может попасть сразу в несколько файлов, если у него несколько классов.

Основной результат:

- `golden_set_by_class/*.csv`
- `golden_set_class_counts_summary.csv`
- `golden_set_saved_files_summary.csv`
- `golden_set_all_classes_stacked.csv`

Файл нужен для создания набора примеров, который можно вручную проверить и использовать как более надежную валидационную выборку.
