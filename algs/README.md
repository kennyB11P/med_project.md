Краткое описание проекта

Проект посвящен анализу отзывов Wildberries и подготовке датасета для multi-label классификации. Один отзыв может относиться сразу к нескольким классам: брак, низкое качество, проблема с размером, доставка, упаковка, положительный отзыв и т.д.

Папка dataset

В этой папке лежат ноутбуки для базовой подготовки данных:

скачивание полного датасета отзывов;
очистка текста;
формирование случайной выборки;
первичная разметка отзывов через LLM;
создание golden set для ручной проверки;
обучение baseline-модели RuBERT для multi-label классификации.

Основная роль папки — подготовить исходные данные, получить первую размеченную выборку и проверить, можно ли на ней обучить классификатор.

Папка clustering

В этой папке лежат ноутбуки для кластеризации отзывов и добора дополнительных примеров:

формирование выборки для HDBSCAN;
построение эмбеддингов отзывов;
кластеризация через UMAP + HDBSCAN;
анализ кластеров через LLM;
добор отзывов из полезных кластеров;
точечный поиск примеров для редких классов через похожесть эмбеддингов.

Основная роль папки — найти группы похожих отзывов, понять, где находятся проблемные и редкие классы, и добрать дополнительные размеченные примеры для улучшения датасета.

Общая логика

Сначала данные скачиваются и очищаются, затем отзывы кластеризуются, кластеры анализируются, и из конкретных кластеров я беру примеры для разметки своих классов - размечаю golden set, после чего с помощью LLM размечаю уже больший набор примеров, с оценкой промпта на golden set. На итоговом датасете обучается baseline-модель RuBERT для автоматической классификации отзывов.

Первый эксперимент с моделью

В эксперименте обучал модель для автоматической разметки отзывов покупателей. Задача модели — получить на вход текст отзыва и определить, к каким классам он относится. Так как один отзыв может содержать сразу несколько признаков, задача решалась как multi-label классификация.

В качестве базовой модели был взят DeepPavlov/rubert-base-cased. Это русскоязычная BERT-модель, которая уже умеет работать с русским текстом и извлекать из него смысловые признаки. В этом первом эксперименте я не дообучал всю модель целиком, а заморозил основной RuBERT-энкодер и обучал только простую классификационную голову сверху. Она преобразует выход RuBERT в вероятности по 13 классам отзывов.

То есть модель работает следующим образом: текст отзыва сначала проходит через RuBERT, который превращает его в числовое представление, затем это представление подается в небольшой классификатор, а на выходе получаются вероятности для каждого класса. После этого для каждого класса применяется свой threshold, и модель решает, какие метки поставить отзыву.

Это baseline-модель: простая начальная версия, с которой удобно сравнивать следующие эксперименты. Дальше качество можно улучшать за счет расширения и балансировки обучающей выборки, дообучения большего числа слоев RuBERT или полного fine-tuning модели.

Анализ первого эксперимента

По динамике обучения видно, что модель действительно начала обучаться: train loss снизился примерно с 0.889 до 0.593, а validation loss — примерно с 0.761 до 0.630. Это показывает, что классификационная голова начала подстраиваться под задачу, но сама модель пока остается очень простой, так как основной RuBERT-энкодер не дообучался.

На validation после подбора threshold модель показала следующие результаты:

micro F1: 0.399;
macro F1: 0.234;
weighted F1: 0.506;
micro precision: 0.253;
micro recall: 0.941;
hamming loss: 0.291;
ROC-AUC micro: 0.853;
PR-AUC micro: 0.396.

Из этих чисел видно, что модель больше склонна находить много возможных классов, чем быть очень точной. Recall получился высоким, то есть модель часто захватывает нужные метки, но precision низкий, поэтому вместе с правильными классами она добавляет много лишних. Для первого baseline это ожидаемо: модель еще грубая и работает скорее как первичный фильтр, а не как точный классификатор.

На validation лучше всего модель справилась с более заметными и представленными классами. Например, для класса «Проблема с упаковкой» F1 получился около 0.70, для «Проблема доставки / получения» — около 0.57, для «Положительный отзыв» — около 0.50, для «Несоответствие описанию» — около 0.43. При этом по редким или отсутствующим в train классам качество оказалось нулевым или очень низким, потому что модель почти не видела таких примеров во время обучения.

На golden set качество оказалось слабее. При стандартном threshold 0.5 micro F1 составил около 0.315, macro F1 — около 0.174. После подбора threshold модель стала находить больше правильных меток, но начала сильнее добавлять лишние классы. Для tuned thresholds на golden set получились примерно такие значения:

ROC-AUC micro: 0.687;
ROC-AUC macro: 0.557;
PR-AUC micro: 0.242;
PR-AUC macro: 0.186;
hamming loss: 0.387.

Также exact match на golden set оказался очень строгим: во всех 761 примерах была хотя бы одна ошибка в полном наборе меток. Это объяснимо так как некоторые классы могут иметь пересекающиеся значения, даже, когда сам размечал испытывал сложности, поэтому далее рассмотрю возможность объединения классов или более четкого разделения.
